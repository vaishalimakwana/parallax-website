sap ui lab

The lab is an innovation hub where skills and knowledge are learnt, retained and shared through innovation and experimentation with User Interface technologies.

 

The Lab provides the technology infrastructure on which research and experiments in new technologies and approaches in user interfaces can happen, allowing the development of new offerings in the SAP UI space and the building of skills in emerging technologies.

ui landscape

The diagram gives an overview of SAP UI WORKS landscape. SAP Business suites like SAP ECC, ISU, Retail, CRM, SCM, HCM form the Application foundation layer of SAP business applications.

 

Gateway, Process Integration (PI) and Sybase Unwired Platform (SUP) provide flexibility and middleware tooling that allow nimble adoption of future UI developments by decoupling the application and UI layers.

The Factory is where our best practice methodology is retained and improved, using a combination of agile and traditional iSAP, to fit with wider program management. The UI factory takes the knowledge gained from the Lab and industrialises the component development, through the use of defined methodology and a scalable team.

Project Methodolgy

Staff Role

The project methodology to deliver UI requirement follows iSAP methodology.

As the diagram shows, the different phases are - project preparation, Blueprinting, Build,Test and Go Live. The prototyping during blueprint is done using Capgemini's Rapid, Design and Visualize (RDV) methodology. Agile methodology is used for the build phase. UI requirements can either be one sprint or can be divided into multiple sprints. After a successful build, the development goes into test phase and after successful testing it goes live.
The Factory has clearly defined roles for staff and a well defined set of standard templates in order to accelerate the delivery and capture improvement in process. The factory shares the repository for reusable components generated with the Lab in order to foster future innovation. The Factory enables efficiency in delivery through the process of continual improvement.
about SALES ORDER APPLICATION
Sales Order is a very important document in business. SAP provides complete sales order processing. But it is complex for a normal user and not possible without SAP knowledge. Sometimes in business it happens that a non-SAP person has to create sales order. In such case, that person has to undergo a training to use SAP for sales order processing.
techno
The application is simple HTML and Javascript based user interface and can be accessed easily from web browser over internet. view demo
As part of custom UI, the UI works offers a Sales Order Application which is zero training, easy to use user interface. The application provides various features like user authentication, order status tracking and creation of new sales order. It also displays order details based on order number or delivery number.
about move in move out
It is a very common scenario in utility companies that customers leave their current houses and terminate the contract, customers move to new houses and create a new contract. The application provides a simple user interface to perform home move out and move in operations for utility company customers.
techno
SAPUI5, Jquery, SAP NW Gateway view demo
The move in move out application takes customer number as an input. It fetches details of the customer and then allows user to perform either move out or move in for that customer. The application either terminates existing contract or creates a new contract based on the operation performed.
about Leave Application
The Leave Application provides easy interface to the user to apply for leave or for approver to approve or reject leave requests.
techno
SAPUI5, SAP NW Gateway view demo
SAP provides leave management functionality as part of standard ESS/MSS component. Leave management is a very basic functionality and it is possible that each user of this functionality may not be familiar with SAP. It requires a training to access Leave Management System provided by SAP. As part of custom UI, UI works offers a Leave Application which is zero training, easy to use user interface for Leave Management System provided by SAP. The application provides various features such as user authetication, leave balance display, leave request overview, submit leave and approve/reject leave requests for approvers.
about Stock Lookup Application
Stock lookup application provides available stock and price details of an article in a particular store or in all stores of a region.
techno
SAPUI5, SAP NW Gateway view demo
SAP provides very powerful retail solution, which is implemented by many organizations across the world. A very common requirement in retail world is to find out stock and price details of a particular article. In a conventional way, user has to logon to SAP and find out the required information through standard SAP screens. UI Works offers a Stock Lookup Application, developed using SAPUI5, which is zero training, easy to use user interface to find out stock details. The application allows user to find out stock and price details of a particular article in a single store or in all stores of a particular country.

about PI Monitor Application
SAP PI (Process Integration) is an integration solution provided by SAP.
techno
HTML5, Jquery, Phonegap, SAP NW Gateway
Message monitoring is a very important feature of SAP PI. It is used in following cases - - To get an overview of message processing. - To track the status of messages - To find errors that have occurred and establish what caused them Normally for message monitoring in SAP PI, one has to either login to SAP PI system or open Runtime Workbench. The PI Monitor Application brings message monitoring to mobile. The application provides features like - search messages, display error details for messages in error and cancel/restart message.

about VMS Application
